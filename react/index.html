WebDev React JS Playground

Features

The right-hand side of this page is built in React JS. It is a "2 page" component. Below are all the features and notes about related structure and files.

  • index.js - The main file.
    • "App Title" is a static class
    • "Page Title" is a Dynamic title
    • Options A,B,C show an array using Mapping and Reduce with Sum function
    • "Show/Hide Form" button opens up a form with fields showing different Form functions
      • Form shows simple Submits and User/Email form shows "required" Email field.
      • The field from Sessions.js demonstrates browser caching a variable (Session, Cookie)
    • "Go to Page 2" link demonstrates React Router to move between React pages.
      • Function Component Example to illustrate how to pass functions between components as props
    • useEffect - Demonstrates Hooks to control State
    • Redux Demo - Uses "Click Me" button to show how Redux manages State
    • Apollo - NOT used. A version exists where Apollo is used to pull in Github GraphQL data but was removed for security (token) reasons. You may see Apollo or Github references commented out.
  • App.js - Pulls in Profile.js component
  • Button.js - Button code and click events / states used in index.js Buttons
  • Button.spec.js - Enzyme tests (Dev only) that check Button code
  • Form.js - Contains Form elements when show/hide form button clicked.
  • Form.spec.js - Enzyme tests (Dev only) that check Form code
    • Includes "Snapshot" test example
  • FormErrors.js - Checks form fields for errors / required fields
  • FunctionComponent.js - Code for Function Component Example on Page 2
  • Page2.js - Code for React "Page 2" using Browser Router
  • Profile.js - Was pulling Profile info from Github but not in use for Token/Security Reasons. See "Apollo" note above.
  • Radio.js - Material UI Radio buttons example
  • Red.js - Red (short for Redux) for use with "Click Me" button Redux example
  • redux.js - Manages State for the Red "click me" example
  • registerServiceWorker.js - Used with serviceWorker - to run background tasks and browser related programs, e.g. cache
  • Results.js - Was mostly to show Apollo / GraphQL function using Github. Mostly disabled in this version but needed as a dependency.
  • serviceWorker.js - to run background tasks and browser related programs, e.g. cache
  • Sessions.js - Related to Seassions part of Form. Demonstrates Storing a variable in the Browser (i.e. Cookie)
  • setupTests.js - Used to activate Enzyme for Dev testing
  • Sum.js - Simple addition Function to demonstrate Enzyme (Dev only) testing
  • sum.test.js - Test scripts for Sum testing
  • Enzyme - When this React App is worked on locally, enabling Enzyme (using 'npm test') will do on-the-fly testing, looking for errors and comparing Snapshots.
